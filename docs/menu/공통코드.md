# 공통코드 메뉴 기능 명세

> 대상 독자: 유지보수 개발자, QA, 운영자
> 목적: 공통코드 화면의 기능/권한/테스트 기준을 단일 문서로 관리

---

## 1. 개요

- 메뉴명: 공통코드
- 경로: `/system/common-code`
- 화면 컴포넌트: `frontend/src/features/commonCode/CommonCodeList.tsx`
- 역할: 대분류/하위 공통코드를 조회, 등록, 수정, 엑셀 다운로드한다.

---

## 2. 주요 기능

- 검색어 기반 목록 조회
- 대분류 펼침/접기 및 하위코드 조회
- 대분류 코드 등록 모달
- 하위 코드 등록 모달
- 그리드 인라인 수정/수정 모달 저장
- 엑셀 다운로드

---

## 3. 버튼/액션 권한

기준 리소스: `system-common-code` -> `system_common_code`

| UI 요소 | 동작 | 권한 코드 |
|---|---|---|
| 대분류코드 등록 버튼 | 등록 모달 열기/등록 | `system_common_code:create` |
| 하위코드 추가 배지 | 하위 등록 모달 열기/등록 | `system_common_code:create` |
| 등록 모달 저장 버튼 | 대분류 등록 submit | `system_common_code:create` |
| 하위등록 모달 저장 버튼 | 하위코드 등록 submit | `system_common_code:create` |
| 코드 링크/수정 진입 | 수정 모달 열기 | `system_common_code:update` |
| 수정 모달 저장 버튼 | 코드 수정 submit | `system_common_code:update` |
| 그리드 셀 편집 | 인라인 수정 | `system_common_code:update` |
| 엑셀다운로드 버튼 | 목록 엑셀 export | `system_common_code:excel_download` |

참고:
- 검색/전체 버튼은 조회 보조 기능으로 별도 액션 권한을 적용하지 않는다.
- submit 핸들러 내부에도 권한 가드가 적용되어 UI 우회 제출을 차단한다.

---

## 4. API 요약

| 메서드 | 경로 | 용도 |
|---|---|---|
| GET | `/api/v1/codes/manage/groups` 계열 | 그룹/상세 조회 |
| PUT | `/api/v1/codes/manage/field` 계열 | 인라인 필드 수정 |
| POST | `/api/v1/codes/manage/group` 계열 | 대분류 등록 |
| POST | `/api/v1/codes/manage/child` 계열 | 하위코드 등록 |
| GET | `/api/v1/codes/manage/export` 계열 | 엑셀 다운로드 |

---

## 5. QA 체크리스트

1. create 권한 없음: 등록 버튼/하위코드 추가/등록 submit이 차단되는가
2. update 권한 없음: 인라인 편집/수정 모달 저장이 차단되는가
3. excel_download 권한 없음: 엑셀 버튼이 숨김 또는 비활성 처리되는가
4. 권한 있어도 submit 중복 클릭 방지가 동작하는가
