# 화주(법인) 정보 메뉴 기능 명세

> **대상 독자**: 유지보수 개발자, QA, 매뉴얼 작성자  
> **목적**: 프로세스·기능 파악, 테스트 시나리오, 사용자 매뉴얼 기초 자료

---

## 1. 개요

- **메뉴명**: 화주(법인) 정보 (Apps > BO — 기초정보)
- **역할**: `OM_CORPORATION_M` 기준 화주(법인) 목록을 검색·조회하고, 그리드에서 일부 필드를 직접 수정하거나 모달로 상세 수정·신규 등록한다.
- **인증**: 로그인(세션) 필요. 미인증 시 API 401, 프론트는 로그인 리다이렉트 처리.
- **UI 정렬**: 목록 툴바는 **주문 관리 > 국내 B2C 주문**과 동일한 패턴(`orders-filter-row`, `orders-search-group`, `FiFilter` / 검색어 / `FiSearch`+검색 / 보조 버튼). 등록·수정 모달은 **주문 수기등록**과 동일한 폼 패턴(`FloatingRow`, 필수 `.required`, `form-control-sm`, 푸터 `product-modal__footer--compact`, `btn-phoenix-secondary`·`btn-phoenix-primary btn-sm`).

---

## 2. 메뉴·화면 구성

| 메뉴(사이드바)   | 경로             | 화면              | 비고 |
| ---------------- | ---------------- | ----------------- | ---- |
| 화주(법인) 정보  | `/basic/shipper` | 화주 목록(그리드) | `pageRegistry` → `ShipperList` |

- **진입**: 사이드바 **BO — 기초정보 > 화주(법인) 정보** 클릭.
- **페이지 헤더 (`PageLayout`)**  
  - 이 화면만 **`showHeaderRefresh={false}`** 로 공통 **새로고침** 버튼을 숨긴다.  
  - 이유: 새로고침은 `invalidateQueries()`로 **앱 전역** 캐시 무효화이고, 목록만 다시 보려면 **검색·전체**로 충분해 역할이 겹치지 않도록 분리.  
  - 다른 메뉴는 기본값으로 새로고침 유지.
- **툴바(그리드 상단, DataGrid `toolbar`)**  
  - **왼쪽** (`input-group input-group-sm orders-search-group`):  
    - 필터 아이콘 래일(`FiFilter` + `orders-toolbar-addon`)  
    - 검색어 입력 (`orders-search-keyword`, placeholder 짧은 문구 i18n `shipper.searchPlaceholder`)  
    - **검색**: 적용 검색어 반영·1페이지로 이동 (`FiSearch`)  
    - **전체**: 입력·적용 검색어 초기화·1페이지 (`FiRotateCcw`)  
  - **오른쪽**: **화주등록** (`FiPlus`), **엑셀다운로드** (`FiDownload`) — `btn-phoenix-secondary btn-sm btn-default-visible`.

## 3. 화면별 기능

### 3.1 화주 목록 — `/basic/shipper`

- **데이터**: `GET /api/v1/corporations/manage` (`keyword`, `page`, `size`). React Query 키 `['corporations','manage', …]`.
- **정렬**: 서버 `created_at DESC`(등록일시 역순). 그리드 컬럼 정렬은 끔(`defaultColDef.sortable: false`).
- **번호 컬럼**: 전역 역순(첫 행=전체 건수 기준 큰 번호), `colId: 'rowNum'`.
- **검색**: 화주코드·화주명·`corporation_info`의 사업자등록번호(`business_no`)·대표전화(`tel_no`)·대표이메일(`email`) ILIKE 부분 일치.
- **컬럼**: 번호, 화주코드, 화주명, 사업자등록번호, 대표전화, 대표이메일, 등록일시.
- **화주코드**: 링크 버튼 → **수정 모달** (`GET /corporations/detail?corporationCd=` → `PUT /corporations/detail`).
- **그리드 직접 편집**  
  - **화주명, 사업자등록번호, 대표전화, 대표이메일** → `PUT /corporations/field`.  
  - 화주명 빈 값 불가(클라이언트에서 원복). 실패 시 SweetAlert + AG Grid `undoCellEditing`.
- **페이징**: `DataGridPaginationFooter`, `pageSize` 50/100/500/1000.
- **엑셀다운로드**: `GET /corporations/export` — 현재 **적용된** 검색어와 동일 조건의 전체 행, 헤더 언어(`lang` / `X-Requested-Lang`).

### 3.2 화주 등록 모달

- **필수**: 화주명만. **화주코드는 입력 없음** — 서버가 `CORP-####`(4자리 zero-pad, 예 `CORP-0021`) 자동 채번. 기존 `CORP-####` 최대값+1 또는 공번 보충. 동시 등록은 DB PK 유일성으로 막고, 중복 시 재채번·재INSERT(제한 횟수). 한도 초과 시 API 메시지 키 `corporations.serial_exhausted`.
- **부가정보**: 수정 모달과 동일 필드 — 사업자번호, 전화, 이메일, 대표자명, 주소, 팩스, 홈페이지 URL, 비고 → JSONB `corporation_info`에 저장.
- **저장 후 계속**: 주 버튼 문구 `shipper.modal.saveAndContinue`. 성공 시 SweetAlert(`registerContinueSuccess`, `{{cd}}`) 후 **폼만 초기화**, 모달은 유지. 상단 법인 목록·관리 목록 쿼리 무효화.
- **닫기**: 헤더 `×`, 하단 **취소**.

### 3.3 화주 수정 모달

- 상세 로드 후 편집. 화주코드는 읽기 전용(`product-modal__input--readonly`).
- **필수**: 화주명(`FloatingRow` + `.required`).
- 저장 시 `PUT /corporations/detail` — 성공 시 모달 닫기 + 목록 무효화.

### 3.4 공통 모달 UI

- `FloatingRow`: 주문 수기등록과 동일 — 필수 시 `form-floating.required`, 라벨에 빨간 `*`, `common.requiredMark`를 `aria-label`에 사용.
- 입력: `form-control-sm`, floating 라벨용 `placeholder=" "`.
- 푸터: `product-modal__footer product-modal__footer--compact`, 취소 `btn-phoenix-secondary btn-sm`, 제출 `btn-phoenix-primary btn-sm`, 제출 중 `common.loading`.

## 4. API 요약

| 메서드 | 경로 | 용도 |
| ------ | ---- | ---- |
| GET | `/api/v1/corporations` | 상단 법인 셀렉트용 활성 목록 |
| GET | `/api/v1/corporations/manage` | 관리 그리드 페이징 (`keyword`, `page`, `size`) |
| GET | `/api/v1/corporations/export` | 검색 조건 엑셀 (`keyword`, `lang`) |
| GET | `/api/v1/corporations/detail` | 모달 상세 (`corporationCd`) |
| PUT | `/api/v1/corporations/field` | 그리드 단건 (`corporationCd`, `field`, `value`) |
| PUT | `/api/v1/corporations/detail` | 모달 일괄 수정 |
| POST | `/api/v1/corporations/register` | 신규 등록(body에 `corporationCd` 없음, 화주명·부가정보) |

**응답 메시지 키(일부)**: `corporations.corporation_nm_required`, `corporations.invalid_field`, `corporations.serial_exhausted`, `corporations.duplicate_cd`(동시성 등 예외 시).

## 5. DB

- **테이블**: `OM_CORPORATION_M`
- **주요 컬럼**: `corporation_cd`(PK), `corporation_nm`, `corporation_info`(JSONB: `business_no`, `tel_no`, `email`, `ceo_nm`, `address`, `fax_no`, `homepage_url`, `remark` 등), `is_active`, `created_at`, `updated_at`, `created_by`, `updated_by`.

## 7. QA 체크리스트

1. 최초 진입 시 목록이 `created_at` 내림차순인지 확인.  
2. 검색어 입력 후 **검색**·Enter → 적용 검색·1페이지, 결과 필터 확인.  
3. **전체** → 검색어 비움·전체 목록·1페이지 확인.  
4. **페이지 헤더에 새로고침 버튼이 없음**(다른 메뉴와 차이) 확인.  
5. 화주코드 클릭 → 수정 모달 로드·저장 → 목록 반영.  
6. 그리드 셀 편집 → API 성공/실패 시 원복·토스트.  
7. **화주등록** → 자동 부여 코드 안내 문구, 저장 후 계속 → 폼 초기화·모달 유지, 상단 법인 드롭다운에 반영.  
8. **엑셀다운로드** 파일 행 수·검색 조건 일치.  
9. 툴바/모달 스타일이 주문(국내 B2C)·수기등록과 시각적으로 일관한지 확인.
