# 에러이력 메뉴 기능 명세

> 대상 독자: 유지보수 개발자, QA, 운영자
> 목적: 현재 구현 상태(placeholder)와 향후 확장 기준을 명확히 관리

---

## 1. 개요

- 메뉴명: 에러이력
- 경로: `/log/error`
- 페이지 등록: `frontend/src/pages/pageRegistry.tsx` (placeholder)
- 현재 상태: 준비 중 화면(실제 목록/조회 기능 미구현)

---

## 2. 현재 동작

- 메뉴 진입 시 "준비 중인 화면입니다." 안내를 표시한다.
- 데이터 조회/수정/다운로드 액션은 제공하지 않는다.

---

## 3. 권한 기준

기준 리소스: `log-error` -> `log_error`

| UI 요소 | 동작 | 권한 코드 |
|---|---|---|
| 메뉴 진입 | 화면 접근 | `log_error:view` |

참고:
- 현재 액션 버튼이 없어 `create/update/delete/excel_download/approve` 권한은 미사용이다.
- 기능 구현 시 버튼별 액션 권한 표를 본 문서에 추가한다.

---

## 4. 향후 구현 시 권장 항목

- 목록 조회(기간/서비스/레벨/키워드)
- 상세 조회(요청 정보, 스택/메시지, 컨텍스트)
- 엑셀 다운로드(`log_error:excel_download`)
- 필요 시 상태 변경/확인 처리(`update` 또는 `approve`)

---

## 5. QA 체크리스트

1. 메뉴 진입 시 placeholder 문구가 정상 노출되는가
2. 미구현 기능 버튼이 노출되지 않는가
3. 추후 구현 시 권한 코드가 문서와 코드에서 일치하는가
