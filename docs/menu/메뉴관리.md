# 메뉴관리 메뉴 기능 명세

> 대상 독자: 유지보수 개발자, QA, 운영자
> 목적: 메뉴 트리 편집 화면의 기능/권한/검증 기준 정리

---

## 1. 개요

- 메뉴명: 메뉴관리
- 경로: `/system/menus`
- 화면 컴포넌트: `frontend/src/features/menuManage/MenuManagePage.tsx`
- 역할: 시스템(BO/OM)별 메뉴 트리를 조회하고 메뉴를 생성/수정/삭제한다.

---

## 2. 주요 기능

- 시스템 코드(BO/OM) 선택 후 트리 조회
- 메뉴 선택 시 상세 정보 편집
- 하위 메뉴 추가/최상위 메뉴 추가
- 메뉴 저장(신규/수정)
- 메뉴 삭제

---

## 3. 버튼/액션 권한

기준 리소스: `system-menus` -> `system_menus`

| UI 요소 | 동작 | 권한 코드 |
|---|---|---|
| 하위메뉴 추가 버튼 | 선택 메뉴 하위 신규 폼 시작 | `system_menus:create` |
| 최상위 메뉴 추가 버튼 | 루트 신규 폼 시작 | `system_menus:create` |
| 저장 버튼(신규 모드) | 신규 메뉴 저장 | `system_menus:create` |
| 저장 버튼(수정 모드) | 기존 메뉴 수정 저장 | `system_menus:update` |
| 삭제 버튼 | 선택 메뉴 삭제 | `system_menus:delete` |
| 수정 폼 입력 필드 | 기존 메뉴 항목 편집 | `system_menus:update` |

참고:
- create 권한 없으면 추가 버튼은 렌더링되지 않는다.
- update/delete 권한은 버튼 disabled + 핸들러 가드로 이중 방어된다.

---

## 4. API 요약

| 메서드 | 경로 | 용도 |
|---|---|---|
| GET | `/api/v1/menus/manage/tree` 계열 | 메뉴 트리 조회 |
| POST | `/api/v1/menus/manage` 계열 | 메뉴 생성 |
| PUT | `/api/v1/menus/manage/{menuId}` 계열 | 메뉴 수정 |
| DELETE | `/api/v1/menus/manage/{menuId}` 계열 | 메뉴 삭제 |

---

## 5. QA 체크리스트

1. create 권한 없음: 하위/최상위 추가 버튼이 노출되지 않는가
2. update 권한 없음: 수정 입력/저장이 불가한가
3. delete 권한 없음: 삭제 버튼으로 삭제 실행이 차단되는가
4. 저장/삭제 성공 후 트리와 상세가 즉시 갱신되는가
