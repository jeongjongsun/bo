# 상품 엑셀 대량 업로드

> **하나의 엑셀 파일**로 단일상품·세트상품을 모두 등록할 수 있다.  
> 시트 3개: **Products**(메인), **Units**(단위/바코드 1:N), **SetComponents**(세트 구성품 1:N).  
> **법인코드는 양식에 없으며**, 등록 시 **현재 선택된 법인**으로 일괄 적용된다.

---

## 1. 시트 구성 요약

| 시트명 | 설명 | 1:N |
|--------|------|-----|
| **Products** | 상품 메인 정보. 1행 = 1상품. 단일/세트 구분은 `product_type` | - |
| **Units** | 단위/바코드. 동일 상품에 여러 행 가능 | Products 1 : N |
| **SetComponents** | 세트 구성품. 세트 상품당 여러 구성품 | Products(SET) 1 : N |

- **법인**: 양식에는 법인코드 컬럼이 **없다**. 등록 시 상단에서 선택한 법인으로 3개 시트 모두 처리.
- **Products**에 없는 상품은 등록되지 않는다. **Units**·**SetComponents**는 **product_cd**로 연결(같은 법인 전제).
- **SetComponents**의 구성품(`component_product_cd`)은 **같은 엑셀의 Products**에 있거나 **이미 DB에 있는** 상품이어야 한다.

---

## 2. 양식 다운로드·헤더 언어

- **양식 다운로드** 시 **현재 선택된 언어**(한글/영어)로 헤더가 생성된다. (GET `/api/v1/products/import-template?lang=ko` 또는 `lang=en`)
- 헤더에 **입력 가능한 값**이 필요한 컬럼은 괄호로 안내한다.
  - **product_type**: `상품유형 (SINGLE 또는 SET)` / `Product Type (SINGLE or SET)` → **SINGLE**, **SET** 외 값은 **SINGLE**로 처리.
  - **is_sale**, **is_display**, **is_base_unit**: `판매여부 (Y 또는 N)` 등 → **Y**, **N** 외 값은 기본값(판매/전시 true, 기본단위 false)으로 처리.
- **숫자 컬럼**(원가, 공급가, 수량 등)에 **문자가 입력된 경우** 해당 셀은 **숫자로 해석되지 않고 무시**된다(해당 필드만 누락, 행 자체는 등록).

---

## 3. 시트별 컬럼 정의 (법인코드 없음)

### 3.1 시트 "Products"

첫 행은 **헤더**(선택 언어에 따라 한글/영문). 2행부터 데이터. **법인코드 컬럼 없음.**

| 컬럼(영문 키) | 필수 | 타입 | 입력 가능한 값 / 비고 |
|---------------|------|------|------------------------|
| product_cd | O | 문자열 | 상품코드 (법인 내 유일) |
| product_nm | O | 문자열 | 상품명 |
| product_type | O | 문자열 | **SINGLE** 또는 **SET**. 그 외는 SINGLE 처리 |
| base_unit_cd | - | 문자열 | 기본단위코드 (EA 등) |
| is_sale | - | Y/N | **Y** 또는 **N**. 그 외는 Y(판매) 처리 |
| is_display | - | Y/N | **Y** 또는 **N**. 그 외는 Y(전시) 처리 |
| product_en_nm ~ remark | - | 문자열/숫자 | 부가정보. 숫자 컬럼에 문자가 들어오면 해당 필드 무시 |

### 3.2 시트 "Units"

| 컬럼 | 필수 | 타입 | 입력 가능한 값 / 비고 |
|------|------|------|------------------------|
| product_cd | O | 문자열 | Products의 product_cd |
| unit_cd | O | 문자열 | 단위코드 (EA, CS, PLT 등) |
| barcode | - | 문자열 | 바코드 |
| pack_qty | - | 정수 | 입수량. 기본 1. 문자가 들어오면 무시 후 기본값 |
| is_base_unit | - | Y/N | **Y** 또는 **N**. 그 외는 N 처리 |

### 3.3 시트 "SetComponents"

| 컬럼 | 필수 | 타입 | 설명 |
|------|------|------|------|
| parent_product_cd | O | 문자열 | 세트 상품의 상품코드 |
| component_product_cd | O | 문자열 | 구성품 상품코드 (같은 엑셀 Products 또는 기등록 상품) |
| component_qty | - | 정수 | 수량. 기본 1 |

---

## 4. 등록 시 법인 적용

- 업로드 시 **현재 선택된 법인**이 **3개 시트 모두**에 적용된다.
- Products·Units·SetComponents의 모든 행이 해당 법인으로 INSERT된다. 양식에 법인코드 컬럼을 넣지 않는다.

---

## 5. 중복 상품 처리 (이미 등록된 경우)

- 엑셀에 10건 입력했는데 그 중 **3건이 이미 해당 법인에 등록된 상품코드**인 경우:
  - **그 3건은 등록하지 않고 스킵**한다.
  - **나머지 7건만** INSERT한다.
  - 응답으로 **successCount(등록 건수)**, **skippedCount(스킵 건수)**, **skippedProductCodes(스킵된 상품코드 목록)** 를 준다.
  - 화면에는 예: "7건 등록되었습니다. (3건 중복으로 스킵)" 형태로 표시한다.

---

## 6. 엑셀 등록 모드 (상품 일괄 / 단위·바코드만 / 세트 구성만)

화면에서 **엑셀 등록 모드**를 선택한 뒤 파일을 업로드한다.

| 모드 | 설명 | 필수 시트 | 용도 |
|------|------|-----------|------|
| **상품 일괄 등록** | 상품 + 단위/바코드 + 세트구성 한 번에 | Products | 신규 상품 등록. Units·SetComponents는 선택. |
| **단위/바코드만 등록** | 이미 등록된 상품에 단위·바코드만 추가 | Units | 상품은 이미 DB에 있어야 함. product_cd는 **선택 법인에 등록된 상품코드**. |
| **세트 구성만 등록** | 이미 등록된 세트 상품에 구성품만 추가 | SetComponents | 부모·구성품 상품이 **선택 법인에 이미 등록**되어 있어야 함. |

- **단위/바코드만**: Products 시트 없이 **Units** 시트만 채워 업로드. 각 행의 `product_cd`가 해당 법인에 존재하는 상품이어야 하며, 없으면 오류(해당 상품 먼저 등록 필요).
- **세트 구성만**: Products 시트 없이 **SetComponents** 시트만 채워 업로드. `parent_product_cd`, `component_product_cd` 모두 해당 법인에 존재하는 상품이어야 함. 동일 (부모, 구성품) 쌍은 중복 시 스킵(INSERT 시 무시).

---

## 7. 업로드 프로세스

- **모드별 검증**
  - **상품 일괄**: 시트 "Products" 필수. "Units", "SetComponents"는 없거나 비어 있으면 무시.
  - **단위/바코드만**: 시트 "Units" 필수. product_cd는 선택 법인 기준으로 DB 조회 후 없으면 오류.
  - **세트 구성만**: 시트 "SetComponents" 필수. parent_product_cd, component_product_cd는 선택 법인 기준으로 product_id 조회 후 없으면 오류.
- **상품 일괄(full)** 모드: Products 시트 처리(중복 스킵·신규만 INSERT) → Units 시트(상품별 기존 단위 DELETE 후 해당 상품 행만 INSERT) → SetComponents 시트(상품별 기존 구성 DELETE 후 해당 세트 행만 INSERT). 응답: successCount, skippedCount, skippedProductCodes.
- **단위/바코드만** 모드: Units 행만 OM_PRODUCT_UNIT에 bulk INSERT(상품별 기존 단위 삭제 후 일괄 등록). 응답: successCount.
- **세트 구성만** 모드: SetComponents 행만 OM_PRODUCT_SET_COMPONENT에 bulk(상품별 기존 구성 삭제 후 일괄 등록). 응답: successCount.

---

## 8. 템플릿 파일

- 상품 목록 툴바의 **양식 다운로드** 버튼으로 **현재 언어**에 맞는 헤더의 양식 엑셀 다운로드 (GET `/api/v1/products/import-template?lang=ko` 또는 `lang=en`).
- product_type, is_sale, is_display, is_base_unit 헤더에는 입력 가능한 값(SINGLE/SET, Y/N)이 괄호로 표기되어 있다.

---

## 9. 성능·제한

- **Bulk INSERT**로 상품·단위·세트구성품을 각각 multi-row INSERT.
- 권장: 한 파일 **최대 2,000~5,000행(Products)** 수준. 서버에서 행 수 상한을 둘 경우 문서에 명시.
- 목록·전체 엑셀 다운로드는 법인별 **전체 조회**(페이징 없음). 대량 데이터 시 서버 메모리·응답 시간 주의.

---

## 10. 관련 API

- **양식 다운로드**: GET `/api/v1/products/import-template?lang=ko|en`
- **일괄 등록**: POST `/api/v1/products/bulk-import` (multipart: file, corporationCd, mode)
