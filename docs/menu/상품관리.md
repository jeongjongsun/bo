# 상품관리 메뉴 기능 명세

> **대상 독자**: 유지보수 개발자, QA, 매뉴얼 작성자  
> **목적**: 프로세스·기능 파악, 테스트 시나리오, 사용자 매뉴얼 기초 자료

---

## 1. 개요

- **메뉴명**: 상품 관리 (Apps > 상품 관리)
- **역할**: 법인별 상품 마스터를 조회·**등록**·수정하고, 단일/세트 상품의 기본정보·부가정보·단위/바코드·세트 구성품을 관리한다.
- **인증**: 로그인(세션) 필요. 미인증 시 API 401, 프론트는 로그인 리다이렉트 처리.

---

## 2. 메뉴·화면 구성


| 메뉴(사이드바)   | 경로                  | 화면         | 비고                                        |
| ---------- | ------------------- | ---------- | ----------------------------------------- |
| 상품 정보      | `/product`          | 상품 목록(그리드) | 목록 조회·**상품등록**·셀 편집·상세 진입                 |
| (상품 상세/편집) | `/product/edit`     | 상품 편집(탭)   | 탭으로 열림. productId 있으면 수정, mode=create면 등록 |
| 사은품 지급 설정  | `/product/gift`     | 준비 중       | placeholder                               |
| 매칭 정보      | `/product/matching` | 준비 중       | placeholder                               |


- **진입**: 사이드바 **상품 관리 > 상품 정보** 클릭 → 목록 화면. **상품등록** 버튼 클릭 → 등록 탭 오픈. 목록에서 **상품코드** 클릭 → 상품 편집 탭 오픈.
- **탭**: 동일 상품을 여러 탭으로 열 수 있음. **목록** 버튼으로 탭만 닫고, **저장**으로 수정/등록 반영.
- **목록 툴바 그룹**: **상품등록** 그룹(수기 등록, 그룹 등록 드롭다운: 양식 다운로드·상품 일괄 등록), **엑셀 다운로드** 그룹(그리드 정보 다운로드, 전체 정보 다운로드). **상품유형 필터**: 전체 / 단일상품(SINGLE) / 셋트상품(SET).

## 3. 화면별 기능

### 3.1 상품 정보 (목록) — `/product`

- **표시**: 현재 선택된 **법인** 기준 상품 목록. **정렬**: product_info.sort_order DESC(동일 시 created_at DESC, product_id). 번호 컬럼은 역순 표시(첫 행=전체 건수). AG Grid, 페이징(50/100/500/1000/5000), 엑셀 내보내기 파일명 `products`.
- **툴바**
  - **상품등록 그룹**: 수기 등록 버튼, 그룹 등록 드롭다운(양식 다운로드, 상품 일괄 등록).
  - **엑셀 다운로드 그룹**: 그리드 정보 다운로드(현재 그리드 데이터), 전체 정보 다운로드(OM_PRODUCT_M 전체 컬럼 엑셀, lang 적용).
- **상품유형 필터**: 전체 / 단일상품(SINGLE) / 셋트상품(SET). 데이터 있는 시트만 필터링해 표시.
- **컬럼**: 순번, **상품코드**(클릭 시 편집 탭 오픈), 상품명, 상품유형(SINGLE/SET 뱃지), 기본단위, 판매여부, 전시여부, 생성자, 생성일시.
- **편집 가능(셀 편집)**  
  - **상품명, 판매여부, 전시여부**: 그리드에서 직접 수정 후 **필드 단건 API** 호출.  
  - 상품유형·기본단위는 API 지원하나 현재 그리드에서는 비편집(편집 탭에서만 변경 가능).  
  - 실패 시 토스트 에러 + 그리드 해당 셀 원복.
- **필수 조건**: 상단 법인 선택이 되어 있어야 목록 API 호출(`corporationCd` 필수). **상품등록** 시에도 법인 선택 필수.

### 3.2 상품 편집 / 상품 등록 — `/product/edit`

- **진입**
  - **수정**: 목록에서 상품코드 클릭 시 `productId`를 탭 데이터로 넘겨 편집 화면 로드.
  - **등록**: 목록 툴바 **상품등록** 클릭 시 `data: { mode: 'create' }` 로 동일 경로 오픈. 법인 미선택 시 "법인을 선택한 후 목록을 조회해 주세요" 안내.
- **로드 실패**: 수정 모드에서 productId 없음/404 시 에러 메시지 표시.

**탭 구성**

1. **기본정보 (Basic)**
  - **수정 시**: 상품ID(읽기 전용), 상품코드*, 상품명*, 상품유형*(SINGLE/SET), 판매여부, 전시여부.  
  - **등록 시**: 상품ID는 "등록 시 자동 부여" 표시, 나머지 동일(상품코드*, 상품명*, 상품유형* 등).  
  - `*`: 필수 입력(DB NOT NULL 대응).
2. **부가정보 (Info)**
  - 상품 영문명, 카테고리코드, 브랜드코드, 원가, 공급가, 과세유형(TAXABLE/TAX_FREE/ZERO), 안전재고·최소/최대 주문수량, 정렬순서, 상품설명, 이미지URL, 비고.
3. **단위/바코드 (Units)**
  - 테이블: 단위코드(공통코드 PACK_UNIT 연동), 바코드, 패키지수량, 기본단위(Y/N).  
  - 행 추가/삭제. 저장 시 해당 상품의 단위 테이블 **전체 교체**(DELETE 후 INSERT).
4. **세트 구성품 (Set Components)**
  - **상품유형이 SET일 때만** 탭 노출.  
  - 테이블: 구성 상품(검색 가능한 단일 선택), 수량.  
  - 구성 상품 선택은 Choices.js 기반 검색 셀렉트(상품코드·상품명 검색). 자기 자신은 목록에서 제외.  
  - 행 추가/삭제. 저장 시 해당 상품의 세트 구성 테이블 **전체 교체**.

**공통**

- **저장**
  - **수정 모드**: 폼 제출 시 **상품 수정 API** PUT `/api/v1/products/{productId}` 호출(기본·부가·단위·세트구성 일괄). 성공 시 성공 토스트 + 상품 쿼리 무효화.
  - **등록 모드**: 폼 제출 시 **상품 등록 API** POST `/api/v1/products` 호출(body에 `corporationCd` 필수). 성공 시 성공 토스트 + 탭 닫기 + 목록 쿼리 무효화.
- **목록**: 탭만 닫고 목록 화면으로 돌아감(저장 여부와 무관).
- **단위 코드**: 공통코드(PACK_UNIT) 조회, 1시간 캐시. 없으면 EA 등 기존값 유지.

## 4. 프로세스 요약

1. **목록 보기**: 법인 선택 → 상품 목록 API(GET `/api/v1/products?corporationCd=xxx`) → 그리드 표시.
2. **목록에서 셀 수정**: 상품명/유형/판매/전시 변경 → PUT `/api/v1/products/field` (productId, field, value) → 성공 시 목록 쿼리 무효화.
3. **상세 진입**: 상품코드 클릭 → 탭 오픈(path `/product/edit`, data.productId) → 상세 API(GET `/api/v1/products/{productId}`) + 단위코드 API → 폼·단위·세트구성 표시.
4. **편집 저장**: 폼 + 단위/세트 테이블 상태 → PUT `/api/v1/products/{productId}` (일괄) → 성공 시 상품 관련 쿼리 무효화.
5. **등록**: 상품등록 버튼 → 탭 오픈(create 모드) → 폼 입력 → POST `/api/v1/products` (corporationCd + 폼 데이터) → 성공 시 탭 닫기, 목록 갱신.
6. **전체 정보 엑셀 다운로드**: GET `/api/v1/products/export-full?corporationCd=xxx&lang=ko|en` (또는 헤더 X-Requested-Lang, Accept-Language). 현재 선택 언어에 따라 헤더 한글/영문.

## 5. 엑셀 일괄 등록 프로세스(상세)

엑셀 대량 업로드 시 **3개 시트(Products, Units, SetComponents)**를 모두 확인해 **데이터가 있는 시트만** 아래 순서대로 처리한다. 상세 양식·컬럼은 [상품관리_엑셀업로드.md](상품관리_엑셀업로드.md) 참고.

### 5.1 처리 순서

1. **Products 시트** (데이터 있으면)
2. **Units 시트** (데이터 있으면)
3. **SetComponents 시트** (데이터 있으면)

- Products 시트가 **없거나 비어 있어도** 오류가 아니며, Units/SetComponents만 있으면 해당 시트만 위 순서대로 처리한다.

### 5.2 가. Products 시트

- **데이터가 있을 때만** 실행.
- 검증: 필수값(product_cd, product_nm). 엑셀 내 product_cd 중복 시 오류.
- 선택 법인 + product_cd가 **이미 DB에 있으면** 해당 행은 스킵(등록하지 않음). 스킵된 상품코드는 결과의 `skippedProductCodes`로 반환.
- **신규만** OM_PRODUCT_M에 INSERT. 이번에 등록한 상품에 대해 (corporationCd + "\t" + productCd) → productId 맵을 구성해 이후 Units/SetComponents에서 사용.

### 5.3 나. Units 시트

- **데이터가 있을 때만** 실행. (Products 처리 이후)
- 각 행의 (법인코드, product_cd)로 **product_id 조회**: 먼저 위에서 만든 맵(이번에 등록한 상품 + 스킵된 기존 상품의 productId)에서 찾고, 없으면 DB 조회 후 맵에 캐시. **product_id가 없으면 해당 행만 스킵**(에러 아님).
- **상품별로** 그룹핑한 뒤, **각 상품에 대해**:
  - 해당 상품의 기존 단위 **전부 삭제** (`OM_PRODUCT_UNIT` delete by product_id)
  - 해당 상품에 대한 엑셀 행만 **일괄 INSERT** (MERGE 아님).

### 5.4 다. SetComponents 시트

- **데이터가 있을 때만** 실행. (Products 처리 이후)
- 각 행의 parent(법인코드, parent_product_cd), component(법인코드, component_product_cd)로 **product_id 조회**(동일 맵 + DB 캐시). **parent_id 또는 component_id가 없으면 해당 행만 스킵**(에러 아님).
- **parent_id(세트 상품)별로** 그룹핑한 뒤, **각 세트 상품에 대해**:
  - 해당 세트의 기존 구성 **전부 삭제** (`OM_PRODUCT_SET_COMPONENT` delete by product_id)
  - 해당 세트에 대한 엑셀 행만 **일괄 INSERT** (MERGE 아님).

### 5.5 공통·반환

- 한 트랜잭션으로 처리. 실패 시 전부 롤백.
- 반환: 등록된 상품 수(successCount), 스킵된 상품 수(skippedCount), 스킵된 상품코드 목록(skippedProductCodes).

## 6. API 요약


| 메서드  | 경로                                 | 용도            | 비고                                                                                               |
| ---- | ---------------------------------- | ------------- | ------------------------------------------------------------------------------------------------ |
| GET  | `/api/v1/products`                 | 상품 목록         | 쿼리: corporationCd(필수). 정렬: sort_order DESC, created_at DESC                                      |
| GET  | `/api/v1/products/export-full`     | 전체 컬럼 엑셀 다운로드 | 쿼리: corporationCd(필수), lang(선택). 헤더 X-Requested-Lang, Accept-Language 가능                         |
| GET  | `/api/v1/products/import-template` | 업로드 양식 엑셀     | 쿼리: lang(선택). 헤더 한글/영문                                                                           |
| GET  | `/api/v1/products/{productId}`     | 상품 상세         | 404 시 에러                                                                                         |
| POST | `/api/v1/products`                 | 상품 등록         | body: corporationCd 필수 + 기본·부가·units·setComponents. 201 + ProductDetail                          |
| PUT  | `/api/v1/products/{productId}`     | 상품 수정(일괄)     | body: 기본·부가·units·setComponents. 단위/세트는 전체 교체                                                    |
| PUT  | `/api/v1/products/field`           | 필드 단건 수정      | body: productId, field, value (productCd, productNm, productType, baseUnitCd, isSale, isDisplay) |
| POST | `/api/v1/products/bulk-import`     | 엑셀 일괄 등록      | multipart: file, corporationCd, mode(full|unitsOnly|setOnly)                                     |


- **인증**: 세션 `userId` 없으면 401, `ApiResponse.fail(ERR_SESSION_EXPIRED)`.
- **에러**: 상품코드 중복(동일 법인) 시 200 + `ERR_CONFLICT` + 메시지. 잘못된 필드/404 시 400/404 + `ERR_BAD_REQUEST`/`ERR_NOT_FOUND`.

## 7. DB·테이블 참고

### 7.1 상품 ID 생성 규칙

- **규칙**: 신규 등록 시 `product_id`는 서버에서 자동 생성한다. 형식은 `**PRD-` + UUID 32자(hex, 하이픈 제거)**.
- **예시**: `PRD-a1b2c3d4e5f6789012345678abcdef12`
- **적용**: `ProductService.createProduct()` 내에서 생성 후 INSERT. 컬럼 `OM_PRODUCT_M.product_id` VARCHAR(48)에 맞춤.
- **비고**: 사용자 입력이 아니며, 등록 폼에는 "등록 시 자동 부여"로 표시한다.

### 7.2 테이블·스크립트

- **메인**: `OM_PRODUCT_M` (docs/ddl/om_product_m.sql).  
  - 필수 컬럼: product_cd, product_nm, product_type (편집 폼 필수 표시와 동일).  
  - 부가정보: product_info JSONB(상품 영문명, 카테고리, 원가, 공급가, 과세유형 등).  
  - **목록 정렬**: 상품 목록·전체엑셀 조회 시 `(product_info->>'sort_order')::int DESC NULLS LAST`, `created_at DESC`, `product_id`. 인덱스는 DDL 및 docs/ddl/om_product_indexes_recommended.sql 참고.
- **단위/바코드**: `OM_PRODUCT_UNIT` (docs/ddl/om_product_unit.sql). product_id 기준 삭제 후 일괄 INSERT. 기본단위 조회 시 (product_id, is_base_unit) 조건.
- **세트 구성품**: `OM_PRODUCT_SET_COMPONENT` (docs/ddl/om_product_set_component.sql). product_id 기준 삭제 후 일괄 INSERT.
- **공통코드**: 단위 목록은 코드 도메인 `PACK_UNIT` (CodeService/캐시).
- **성능·인덱스**: 목록/export·법인+상품코드 조회·단위 기본단위 조회 최적화용 인덱스는 **docs/ddl/om_product_indexes_recommended.sql** 참고(선택 적용).
- **데이터·스크립트**: 테이블 정의(DDL)는 `docs/ddl/`. 상품 시드/임시 데이터 등 **데이터 생성용 SQL(DML)**은 `**docs/dml/`** 폴더에 저장한다 (프로젝트 DB 규칙: docs/08-DB-표준.md §7.6).

## 8. QA 체크리스트 (테스트 시나리오)

- **목록**  
  - 법인 미선택 시 목록 미조회(또는 빈 목록).  
  - 법인 선택 시 해당 법인 상품만 표시. **정렬**: sort_order DESC, created_at DESC(동일 시 product_id). 번호는 역순(첫 행=전체 건수).  
  - 상품유형 필터: 전체/단일상품/셋트상품. 상품코드 클릭 시 편집 탭 오픈, 동일 상품 여러 탭 오픈 가능.
- **목록 셀 편집**  
  - 상품명/상품유형/판매여부/전시여부 수정 후 자동 저장.  
  - 상품코드 중복 입력 시 에러 메시지 및 셀 원복.  
  - 네트워크 실패 시 에러 메시지 및 셀 원복.
- **편집 기본정보**  
  - 필수값(상품코드, 상품명, 상품유형) 비워두고 저장 시 HTML5 validation 또는 서버 에러.  
  - 상품코드 중복 시 서버 에러 메시지 표시.
- **편집 부가정보**  
  - 숫자 필드(원가, 공급가, 수량 등) 숫자만 입력.  
  - 과세유형 선택 저장 반영.
- **편집 단위/바코드**  
  - 행 추가/삭제 후 저장 시 DB에 반영(기존 행 삭제 후 입력 행만 존재).  
  - 기본단위 1개만 Y 허용(UI/서버 동작 확인).  
  - 단위코드는 공통코드 목록과 일치.
- **편집 세트 구성품**  
  - 상품유형 SET일 때만 탭 표시.  
  - 구성 상품 검색·선택, 수량 변경, 행 추가/삭제 후 저장 시 DB 반영.  
  - 자기 자신 선택 불가(목록에서 제외).  
  - 수량 1 미만 방지(최소 1).
- **상품 등록**  
  - 법인 미선택 시 상품등록 탭에서 "법인을 선택한 후 목록을 조회해 주세요" 표시.  
  - 상품등록 버튼 → 등록 탭 오픈, 기본정보(상품코드·상품명·상품유형 필수)·부가정보·단위/세트구성 입력 후 저장 → POST 호출, 성공 시 탭 닫기·목록 갱신.  
  - 상품코드 중복(동일 법인) 시 에러 메시지. product_id는 저장 후 자동 부여(표시만 "등록 시 자동 부여").
- **공통**  
  - 저장 성공 시 성공 토스트.  
  - 401 시 로그인 페이지 리다이렉트.  
  - 존재하지 않는 productId로 편집 진입 시 404/에러 메시지.

## 10. 매뉴얼 작성 시 참고

- 사용자 관점에서는 **상품 정보** 메뉴에서 목록을 보고, **상품코드**를 눌러 해당 상품을 편집한다고 안내하면 됨.
- **세트 상품**은 상품유형을 SET로 두고, **세트 구성품** 탭에서 다른 상품과 수량을 지정해 구성한다고 설명 가능.
- **단위/바코드**는 판매 단위별로 바코드·패키지 수량·기본단위 여부를 등록하는 용도로 안내.
