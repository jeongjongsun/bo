# 환경설정 메뉴 기능 명세

> **대상 독자**: 유지보수 개발자, QA, 매뉴얼 작성자  
> **목적**: 사용자별 환경설정 기능 파악, 테스트 시나리오, 사용자 매뉴얼 기초 자료

---

## 1. 개요

- **메뉴명**: 환경설정 (Pages > 환경설정)
- **역할**: 로그인한 사용자별 환경설정을 조회·저장한다. 주문 간편 보기, 기본 법인(화주사) 설정을 지원한다.
- **인증**: 로그인(세션) 필요. 본인 설정만 조회·저장 가능.

---

## 2. 메뉴·화면 구성

| 메뉴(사이드바) | 경로 | 화면 | 비고 |
|----------------|------|------|------|
| 환경설정 | `/settings` | 환경설정 폼 | 개인정보(이름·비밀번호), 주문 간편 보기, 기본 법인 |

- **진입**: 사이드바 **Pages > 환경설정** 클릭 → 환경설정 화면.

---

## 3. 화면별 기능

### 3.1 환경설정 — `/settings`

- **개인정보 (상단 카드)**
  - **이름**: 로그인 사용자 표시명. `GET /api/v1/profile`로 조회 후 표시.
  - **새 비밀번호 / 새 비밀번호 확인**: 두 필드로 오입력 방지. **비워 두면 비밀번호는 변경하지 않음** (안내 문구 표시).
  - **비밀번호 복잡도**: 8자 이상, 영문 1자 이상(대소문자 구분 없음)·숫자·특수문자 각 1자 이상. 안내 문구 및 백엔드 검증 적용.
  - **저장** 클릭 시 `PUT /api/v1/profile` 호출. 이름 변경 시 세션의 표시명도 갱신. 성공 시 "개인정보가 저장되었습니다.", 실패 시 에러 메시지(불일치·복잡도 위반 등).
- **환경설정 (하단 카드)**
  - **주문 간편 보기**: 스위치(체크박스). 활성화 시 주문 그리드에 최소 항목만 표시.
  - **기본 법인(화주사)**: 법인 목록 드롭다운. "선택 안 함" 또는 특정 법인 선택.
  - **주문 그리드 기간 검색 기본값**: 주문일 / 등록일 드롭다운. 저장 시 주문 목록 진입 시 기간 검색 조건의 기본값으로 사용.
  - **저장** 클릭 시 `PUT /api/v1/user-settings` 호출. 성공 시 "환경설정이 저장되었습니다.", 실패 시 에러 메시지.
- **초기 로드**: `GET /api/v1/profile`, `GET /api/v1/user-settings`로 조회. 없으면 기본값 적용.

## 4. 환경설정 항목 상세

### 4.1 주문 간편 보기

- **의미**: 활성화 시 주문 관리 그리드에서 최소 컬럼만 표시.
- **최소 표시 컬럼**: 체크박스(선택), 순번, 주문번호, 합포장번호, 상점명, 주문일, 등록일, 송장번호, 상품코드, 상품명, 수량, 금액, 수취인.
- **탭별 규칙(간편 보기에서도 동일 적용)**  
  - **합포장번호**: 합포장처리 탭부터 표시(주문접수 탭에서는 숨김).  
  - **송장번호**: 발송준비 탭부터 표시(발송준비 이전 탭에서는 숨김).
- **API 최소 필드**: 간편 보기가 켜져 있으면 목록 API에 `minimalColumns=true`가 전달되어, 서버가 최소 컬럼만 SELECT·응답하므로 **로딩 시간·응답 용량이 줄어듭니다**.
- **비고**: 설정 변경 후 주문 관리 화면으로 이동하면 즉시 반영(React Query 캐시 무효화로 최신 설정 사용).

### 4.2 기본 법인(화주사)

- **의미**: 로그인 후 상단 법인 선택 드롭다운의 기본값.
- **동작**: 앱 로드·로그인 후 법인 목록을 불러온 뒤, 환경설정에 저장된 기본 법인이 있으면 해당 법인을 선택하고, 없으면 목록의 첫 번째 법인을 선택한다.
- **선택 안 함**: 기본 법인을 설정하지 않으면 첫 번째 법인이 기본으로 선택된다.

### 4.3 주문 그리드 기간 검색 기본값

- **의미**: 주문 목록(그리드)의 기간 검색에서 사용할 **기준**의 기본값(주문일 vs 등록일).
- **동작**: 환경설정에서 "주문일" 또는 "등록일"을 선택해 저장하면, 주문 관리 메뉴 진입 시 그리드 상단의 기간 검색 드롭다운과 실제 API 검색 조건(dateType)이 해당 값으로 설정된다. 저장값이 없거나 null이면 주문일(ORDER_DT)이 기본.
- **API 연동**: `GET /api/v1/orders`의 `dateType` 파라미터(ORDER_DT | REGIST_DT)와 동일 값.

## 5. API 요약

| 메서드 | 경로 | 용도 |
|--------|------|------|
| GET | `/api/v1/profile` | 현재 사용자 프로필 조회 (이름, emailId, mobileNo) |
| PUT | `/api/v1/profile` | 현재 사용자 프로필 수정 (body: name, newPassword?, newPasswordConfirm?). 비밀번호 미입력 시 변경 안 함. 복잡도·일치 검증 백엔드 적용. |
| GET | `/api/v1/user-settings` | 현재 사용자 환경설정 조회 |
| PUT | `/api/v1/user-settings` | 현재 사용자 환경설정 저장 (body: orderSimpleViewYn, defaultCorporationCd, defaultOrderDateType) |

- **인증**: SessionAuthInterceptor 적용. 401 시 로그인 리다이렉트.
- **주문 목록 연동**: 간편 보기 on 시 `GET /api/v1/orders?minimalColumns=true` 로 호출되어 최소 필드만 응답(용량·로딩 절감).

## 6. DB/테이블 참고

- **테이블**: `om_user_setting_m` (docs/ddl/om_user_setting_m.sql)
- **구조**: user_id(PK), **setting_values(JSONB)**, created_at, updated_at, created_by, updated_by. 설정 항목은 JSONB로 관리해 항목 추가 시 컬럼 변경 없음.
- **setting_values 키**: orderSimpleViewYn(boolean), defaultCorporationCd(string|null), defaultOrderDateType(ORDER_DT|REGIST_DT|null) 등.
- **관계**: user_id → om_user_m(user_id).
- **마이그레이션**: 기존 컬럼 구조 DB는 `docs/ddl/om_user_setting_m_migrate_to_jsonb.sql` 1회 실행 후 구컬럼 제거·JSONB로 전환.
