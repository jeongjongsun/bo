# 감사이력 메뉴 기능 명세

> 대상 독자: 유지보수 개발자, QA, 운영자
> 목적: 감사이력 조회/비교 화면의 기능 및 권한 기준 정리

---

## 1. 개요

- 메뉴명: 감사이력
- 경로: `/log/audit`
- 화면 컴포넌트: `frontend/src/features/auditLog/AuditLogPage.tsx`
- 역할: 시스템/액션/기간/키워드 조건으로 감사 로그를 조회하고 before/after 데이터를 비교한다.

---

## 2. 주요 기능

- 검색 조건(시스템, 액션, 기간, 키워드) 조회
- 조건 초기화
- 목록 새로고침
- 데이터 비교 모달(변경 키 하이라이트)
- 엑셀 다운로드

---

## 3. 버튼/액션 권한

기준 리소스: `log-audit` -> `log_audit`

| UI 요소 | 동작 | 권한 코드 |
|---|---|---|
| 엑셀다운로드 버튼 | 목록 엑셀 export | `log_audit:excel_download` |
| 데이터 조회 배지 | before/after 비교 모달 열기 | 메뉴 접근(`view`) 범위 |
| 검색/전체/목록 새로고침 | 조회 조건 조작 | 메뉴 접근(`view`) 범위 |

참고:
- 엑셀다운로드는 권한 미보유 시 버튼 disabled 및 핸들러 조기 종료로 차단한다.
- 데이터 비교는 조회 기능으로 별도 CUD 액션 권한 없이 메뉴 접근 권한에 종속된다.

---

## 4. API 요약

| 메서드 | 경로 | 용도 |
|---|---|---|
| GET | `/api/v1/audit-logs` 계열 | 감사 로그 목록 조회 |
| GET | `/api/v1/audit-logs/export` 계열 | 엑셀 다운로드 |

---

## 5. QA 체크리스트

1. excel_download 권한 없음: 엑셀 버튼 동작이 차단되는가
2. 검색 조건 변경 시 목록 페이징이 1페이지로 초기화되는가
3. 데이터 비교 모달에서 changed key 하이라이트가 정상 동작하는가
4. 새로고침 버튼으로 쿼리 무효화 후 목록이 갱신되는가
