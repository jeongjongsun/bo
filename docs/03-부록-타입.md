# 개발 표준 부록: 타입 및 코드 예시

**React 입문자**: 이 부록의 타입·코드는 **복사해서 프로젝트에 붙여 넣고, 이름만 바꿔서** 재사용해도 됩니다.

## 1. 공통 응답 타입 (TypeScript / 프론트엔드)

```ts
// API 공통 래퍼
export interface ApiResponse<T> {
  success: boolean;
  data: T | null;
  message: string | null;
  code: string;
}

// 페이징 응답 (data 필드 내용)
export interface PagedData<T> {
  items: T[];
  page: number;
  size: number;
  total: number;
  totalPages: number;
  first: boolean;
  last: boolean;
  sort?: { property: string; direction: 'ASC' | 'DESC' }[];
}

export type PagedResponse<T> = ApiResponse<PagedData<T>>;

// 에러 응답
export interface FieldError {
  field: string;
  message: string;
  rejectedValue?: unknown;
}

export interface ApiErrorResponse {
  success: false;
  data: null;
  message: string;
  code: string;
  errors?: FieldError[] | null;
  timestamp?: string;
  path?: string;
}
```

## 7. 입문자용: 한 화면 완성 흐름 (복사 후 이름만 변경)

아래는 **목록 조회 + 저장 뮤테이션** 한 세트 예시입니다. `users` → `orders` 등으로 바꿔서 재사용하면 됩니다.

### 7.1 API 함수 (`api/users.ts` 또는 `features/users/api.ts`)

```ts
import type { ApiResponse, PagedData } from '@/types';
import { apiClient } from '@/api/client';

export interface User {
  id: string;
  name: string;
  email: string;
}

export interface UserListParams {
  page?: number;
  size?: number;
  sort?: string;
}

export async function fetchUserList(params: UserListParams) {
  const { data } = await apiClient.get<ApiResponse<PagedData<User>>>('/users', { params });
  if (!data.success || !data.data) throw new Error(data.message || '목록 조회 실패');
  return data.data; // { items, page, size, total, totalPages, ... }
}

export async function createUser(body: { name: string; email: string }) {
  const { data } = await apiClient.post<ApiResponse<User>>('/users', body);
  if (!data.success || !data.data) throw new Error(data.message || '저장 실패');
  return data.data;
}
```

### 7.2 훅 (`features/users/hooks.ts`)

```ts
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import { fetchUserList, createUser, type UserListParams } from './api';

export function useUserList(params: UserListParams) {
  return useQuery({
    queryKey: ['users', 'list', params],
    queryFn: () => fetchUserList(params),
  });
}

export function useUserCreate() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: createUser,
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['users', 'list'] });
    },
  });
}
```

### 7.3 화면에서 사용 (`features/users/UserList.tsx`)

```tsx
import { useState } from 'react';
import { useUserList, useUserCreate } from './hooks';

export function UserList() {
  const [page, setPage] = useState(0);
  const { data, isLoading, error } = useUserList({ page, size: 20 });
  const create = useUserCreate();

  if (isLoading) return <div>로딩 중...</div>;
  if (error) return <div>에러: {(error as any)?.response?.data?.message ?? '조회 실패'}</div>;
  if (!data) return null;

  return (
    <div>
      <ul>
        {data.items.map((user) => (
          <li key={user.id}>{user.name} - {user.email}</li>
        ))}
      </ul>
      <p>전체 {data.total}건, {data.page + 1}/{data.totalPages} 페이지</p>
      <button onClick={() => setPage((p) => p - 1)} disabled={data.first}>이전</button>
      <button onClick={() => setPage((p) => p + 1)} disabled={data.last}>다음</button>
      <button
        onClick={() => create.mutate({ name: '홍길동', email: 'h@a.com' })}
        disabled={create.isPending}
      >
        {create.isPending ? '저장 중...' : '추가'}
      </button>
    </div>
  );
}
```

- **데이터**는 `useUserList`가 가져오고, **저장**은 `useUserCreate`가 담당합니다. 화면은 "표시"와 "클릭 시 mutate"만 하면 됩니다.
- 새 기능 추가 시 `User` → `Order`, `users` → `orders` 등으로 바꿔서 같은 구조를 반복하면 유지보수가 쉬워집니다.
